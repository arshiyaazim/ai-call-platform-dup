 8. VOICE SYSTEM ANALYSIS (Fazle Family AI)
================================================================================

ARCHITECTURE:

  The voice system operates as a pipeline:

  ┌─────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐
  │   Caller     │───→│   LiveKit     │───→│  Fazle Voice │───→│ Fazle Brain  │
  │ (Browser/    │    │  (WebRTC     │    │  (Pipeline   │    │  (Reasoning  │
  │  Phone)      │    │   Server)    │    │   Agent)     │    │   Engine)    │
  └─────────────┘    └──────────────┘    └──────────────┘    └──────────────┘
                                              │    │               │
                                         STT  │    │ TTS      Memory│
                                         ▼    ▼    ▼               ▼
                                    ┌─────────┐  ┌──────────┐  ┌────────┐
                                    │ OpenAI  │  │ OpenAI   │  │ Qdrant │
                                    │ Whisper │  │ TTS      │  │(Vector)│
                                    │ (STT)   │  │ (Voice)  │  │        │
                                    └─────────┘  └──────────┘  └────────┘

COMPONENTS:

  1. LIVEKIT SERVER (livekit:7880)
     - WebRTC media server for real-time audio streaming
     - Handles room management, participant tracking, audio routing
     - Proxied via Nginx at livekit.iamazim.com

  2. FAZLE VOICE AGENT (fazle-voice container)
     - LiveKit agents SDK (livekit-agents==0.12.20)
     - VoicePipelineAgent with custom FazleLLM adapter
     - Components:
       * VAD: Silero Voice Activity Detection (preloaded)
       * STT: OpenAI Whisper (real-time speech-to-text)
       * LLM: Custom FazleLLM → routes to Fazle Brain (not direct OpenAI)
       * TTS: OpenAI TTS (text-to-speech, configurable voice: alloy default)
     - Per-session user context: user_id, user_name, relationship
     - Personalized greetings based on relationship type

  3. FAZLE BRAIN (fazle-brain:8200)
     - Core reasoning engine that processes transcribed speech
     - Retrieves relevant memories from Qdrant via Fazle Memory
     - Generates JSON responses with: reply, memory_updates, actions
     - Relationship-aware persona engine (wife/daughter/son/parent/sibling/self)
     - Content safety module (stricter for child accounts)
     - Dual LLM support: OpenAI or Ollama (configurable)

  4. FAZLE MEMORY (fazle-memory:8300)
     - Vector memory storage using Qdrant
     - OpenAI embeddings (text-embedding-3-small, 1536 dimensions)
     - Memory types: preference, contact, knowledge, personal, conversation
     - Semantic search with user isolation (user_id filter)
     - Content-hash deduplication

  5. COTURN (coturn:3478/5349)
     - TURN/STUN server for NAT traversal
     - Enables WebRTC connections behind firewalls
     - Shared-secret authentication with LiveKit

VOICE PIPELINE FLOW:
  1. User opens Fazle UI → requests voice token from /fazle/voice/token
  2. LiveKit room is created, user joins via WebRTC
  3. Fazle Voice agent joins same room (auto-subscribe audio)
  4. User speaks → LiveKit streams audio to Voice agent
  5. Silero VAD detects speech boundaries
  6. OpenAI Whisper transcribes speech to text
  7. Transcribed text sent to FazleLLM adapter
  8. FazleLLM calls Fazle Brain /chat endpoint
  9. Brain retrieves relevant memories, builds persona-aware prompt
  10. Brain calls LLM (OpenAI/Ollama) for reasoning
  11. Brain returns response with reply + memory updates + actions
  12. Voice agent receives reply text
  13. OpenAI TTS converts reply to speech audio
  14. Audio streamed back to user via LiveKit WebRTC

CURRENT LIMITATIONS:
  [!] Voice uses OpenAI APIs exclusively (STT + TTS) — external dependency
  [!] No local/offline STT option (Whisper local not configured)
  [!] No local/offline TTS option (depends on OpenAI)
  [!] No voice cloning — uses generic OpenAI voices (alloy, echo, etc.)
  [!] No real-time streaming for LLM responses (single response, not chunked)
  [!] No call recording or playback functionality
  [!] No phone integration (Twilio SIP) tested end-to-end for Fazle
  [!] min_endpointing_delay=0.5 may cause premature cutoffs in some scenarios
